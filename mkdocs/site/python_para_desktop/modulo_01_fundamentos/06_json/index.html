    O QUE É JSON¶

    2.1 Motivação — Por que Texto Simples Não é Suficiente¶

    No módulo anterior, salvamos listas e textos simples. Funciona, mas tem problemas:

    • Como salvar uma lista de dicionários?
    • Como representar dados aninhados (ex.: cliente com endereço e telefones)?
    • Como garantir que outro sistema (um site, um aplicativo) consiga ler os dados que gravamos?

    A resposta para essas perguntas é um formato chamado JSON (JavaScript Object Notation).

    📌 RESUMINDO

    JSON é um formato leve de intercâmbio de dados, fácil de ler e escrever para humanos, e fácil de interpretar e gerar para máquinas.

    2.2 Explicação — O que é JSON e Por que Usar¶

    JSON é um formato de texto que segue regras específicas. Ele foi criado a partir da sintaxe de objetos do JavaScript, mas hoje é usado em praticamente todas as linguagens de programação.

    Principais características:

    • É baseado em texto (pode ser aberto em qualquer editor).
    • Suporta estruturas aninhadas (objetos dentro de objetos, listas dentro de objetos etc.).
    • É independente de linguagem — Python, Java, C#, PHP, todos conseguem ler e escrever JSON.

    Analogia da Carteira Organizadora

    Imagine que você tem uma carteira com vários compartimentos: Um para cartões Outro para notas Um para moedas Outro para fotos Se você simplesmente jogar tudo em uma gaveta (arquivo de texto simples), fica bagunçado. JSON é como uma carteira organizada: cada item tem seu lugar, e você sabe exatamente onde procurar.

    Onde JSON é usado?

    • Em APIs (sites e aplicativos trocam dados via JSON)
    • Em arquivos de configuração (ex.: package.json do Node.js)
    • Em bancos de dados NoSQL (como MongoDB)
    • Em comunicação entre sistemas (microservices)

    2.3 Comparação — Python vs. JSON¶

    Python e JSON têm sintaxes bem parecidas, mas com diferenças importantes.

    Python JSON
    True true
    False false
    None null
    'texto' (aspas simples) "texto" (só aspas duplas)
    [1, 2, 3] [1, 2, 3] (lista)
    {"nome": "Ana"} (chaves com aspas opcionais) {"nome": "Ana"} (chaves devem ser aspas duplas)
    Chaves podem ser números, strings, etc. Chaves sempre strings (aspas duplas)

    Exemplo de um dicionário Python:

    pessoa = {
    "nome": "Carlos",
    "idade": 30,
    "ativo": True,
    "filhos": None
}

    Mesmo dicionário em JSON (como texto):

    {
    "nome": "Carlos",
    "idade": 30,
    "ativo": true,
    "filhos": null
}

    Perceba que True virou true, None virou null, e tudo está entre aspas duplas.

    🧠 VOCÊ SABIA?

    O nome JSON vem de JavaScript Object Notation , mas é usado com todas as linguagens. O Python tem um módulo chamado json que faz a conversão entre estruturas Python e JSON.

    2.4 Exemplo Pequeno — Convertendo Dicionário para JSON¶

    Vamos usar o módulo json para converter um dicionário Python em uma string JSON.

    import json

pessoa = {
    "nome": "Ana",
    "idade": 28,
    "cidade": "São Paulo",
    "ativo": True
}

# Converter dicionário para string JSON
json_string = json.dumps(pessoa)

print(json_string)
print(type(json_string))

    Saída:

    {"nome": "Ana", "idade": 28, "cidade": "São Paulo", "ativo": true}

    Explicação linha a linha:

    • import json — Importamos o módulo json da biblioteca padrão.
    • json.dumps(pessoa) — dumps significa dump string. Converte um objeto Python (dicionário, lista, etc.) em uma string JSON.
    • A string resultante tem aspas duplas, true em vez de True, e não tem None (seria null).

    2.5 Exemplo Intermediário — Convertendo JSON para Dicionário¶

    Agora o caminho inverso: de uma string JSON para um dicionário Python.

    import json

# Simulando dados recebidos de uma API
json_string = '{"nome": "João", "idade": 35, "cidade": "Rio de Janeiro", "ativo": true}'

# Converter string JSON para dicionário
pessoa = json.loads(json_string)

print(pessoa)
print(type(pessoa))
print(pessoa["nome"])

    Saída:

    {'nome': 'João', 'idade': 35, 'cidade': 'Rio de Janeiro', 'ativo': True}
<class 'dict'>
João

    Explicação:

    • json.loads — load string. Converte uma string JSON em um objeto Python (dicionário, lista, etc.).
    • Os valores true e false viram True e False em Python.
    • null vira None.

    2.6 Exemplo Real — Lista de Objetos em JSON¶

    Muito comum em APIs: uma lista de objetos.

    import json

alunos = [
    {"nome": "Maria", "nota": 8.5},
    {"nome": "José", "nota": 7.0},
    {"nome": "Pedro", "nota": 9.2}
]

# Converter para JSON
json_alunos = json.dumps(alunos, indent=2)   # indent para formatação bonita
print(json_alunos)

    Saída:

    [
  {
    "nome": "Maria",
    "nota": 8.5
  },
  {
    "nome": "José",
    "nota": 7.0
  },
  {
    "nome": "Pedro",
    "nota": 9.2
  }
]

    💡 DICA DE OURO

    Use indent para deixar o JSON legível. Para compactar (economizar espaço), omita o indent .

    2.7 Estruturas Aninhadas — Objetos dentro de Objetos¶

    JSON permite aninhamento profundo:

    import json

pedido = {
    "id": 1001,
    "cliente": {
        "nome": "Fernanda",
        "email": "fernanda@email.com"
    },
    "itens": [
        {"produto": "Notebook", "quantidade": 1, "preco": 2500.00},
        {"produto": "Mouse", "quantidade": 2, "preco": 50.00}
    ],
    "total": 2600.00
}

print(json.dumps(pedido, indent=2))

    2.8 Erros Comuns ao Trabalhar com JSON¶

    Erro Causa Solução
    JSONDecodeError A string JSON está malformada (falta aspas, vírgula etc.) Validar o JSON com um validador online.
    TypeError: Object of type ... is not JSON serializable Tentar serializar um objeto não suportado (ex.: datetime) Converter para string antes ou usar parâmetro default .
    Usar aspas simples no JSON JSON exige aspas duplas Substituir ou usar json.dumps() que já coloca aspas duplas.
    Chave sem aspas No JSON, as chaves devem ser strings com aspas duplas. Sempre usar chaves com aspas duplas.

    ⚠️ ERRO MUITO COMUM

    Ao escrever JSON manualmente, é fácil esquecer uma vírgula ou colocar aspas simples. Prefira usar json.dumps() para gerar JSON automaticamente a partir de estruturas Python.

    2.9 Resumo do Módulo 2¶

    • JSON é um formato de texto para troca de dados.
    • É independente de linguagem e muito utilizado em APIs.
    • Python → JSON: json.dumps()
    • JSON → Python: json.loads()
    • Valores booleanos: True ↔ true, False ↔ false, None ↔ null
    • Sempre use aspas duplas no JSON.
    • O módulo json lida com conversão de forma segura.

    TRABALHANDO COM ARQUIVOS JSON¶

    3.1 Motivação — Indo Além do Texto Puro¶

    Já sabemos salvar texto e listas simples. Agora vamos salvar estruturas completas (listas de dicionários, objetos aninhados) diretamente em arquivos, mantendo a formatação JSON. Assim, podemos carregar o arquivo e recuperar exatamente os mesmos dados sem precisar fazer parsing manual.

    💡 DICA DE OURO

    Usar JSON para salvar dados é muito melhor do que texto personalizado porque:

    • Você não precisa criar um "parser" para interpretar seus dados.
    • Outras linguagens e ferramentas podem ler seus arquivos sem esforço.
    • Os dados ficam organizados e legíveis.

    3.2 Explicação — Salvando e Carregando com json.dump e json.load¶

    Além de dumps e loads (que trabalham com strings), temos dump e load que trabalham diretamente com arquivos.

    Função Entrada Saída
    json.dumps(obj) Objeto Python String JSON
    json.loads(string) String JSON Objeto Python
    json.dump(obj, arquivo) Objeto Python + arquivo aberto para escrita Escreve no arquivo
    json.load(arquivo) Arquivo aberto para leitura Objeto Python

    Analogia da Mochila

    • dumps é como arrumar seus pertences para colocar na mochila (string JSON).
    • dump é como colocar a mochila no porta-malas do carro (arquivo).
    • loads é como tirar os pertences da mochila.
    • load é como pegar a mochila do porta-malas e abri-la.

    3.3 Exemplo Pequeno — Salvando um Dicionário em Arquivo JSON¶

    import json

dados = {
    "nome": "Mariana",
    "idade": 32,
    "hobbies": ["ler", "viajar", "fotografar"]
}

# Salvando em arquivo
with open("dados.json", "w", encoding="utf-8") as arquivo:
    json.dump(dados, arquivo, indent=2, ensure_ascii=False)

print("Arquivo JSON salvo!")

    Explicação:

    • open("dados.json", "w", encoding="utf-8") — Abre o arquivo para escrita com codificação UTF-8 (para suportar acentos e caracteres especiais).
    • json.dump(dados, arquivo, indent=2, ensure_ascii=False) — Escreve o dicionário dados no arquivo, com indentação de 2 espaços e sem converter caracteres Unicode para \uXXXX (mantém acentos legíveis).

    O arquivo dados.json terá o conteúdo:

    {
  "nome": "Mariana",
  "idade": 32,
  "hobbies": ["ler", "viajar", "fotografar"]
}

    3.4 Exemplo Intermediário — Carregando um Arquivo JSON¶

    import json

with open("dados.json", "r", encoding="utf-8") as arquivo:
    dados_carregados = json.load(arquivo)

print(dados_carregados)
print(type(dados_carregados))
print(dados_carregados["nome"])

    Saída:

    {'nome': 'Mariana', 'idade': 32, 'hobbies': ['ler', 'viajar', 'fotografar']}
<class 'dict'>
Mariana

    3.5 Exemplo Real — Persistindo uma Lista de Alunos com JSON¶

    Vamos construir um sistema simples que gerencia uma lista de alunos. Os dados serão salvos em um arquivo JSON.

    import json

# Nome do arquivo onde os dados serão salvos
ARQUIVO_JSON = "alunos.json"

# Função para carregar os dados do arquivo
def carregar_alunos():
    try:
        with open(ARQUIVO_JSON, "r", encoding="utf-8") as f:
            return json.load(f)
    except FileNotFoundError:
        # Se o arquivo não existir, retorna uma lista vazia
        return []
    except json.JSONDecodeError:
        # Se o arquivo estiver corrompido, retorna lista vazia
        return []

# Função para salvar a lista de alunos
def salvar_alunos(alunos):
    with open(ARQUIVO_JSON, "w", encoding="utf-8") as f:
        json.dump(alunos, f, indent=2, ensure_ascii=False)

# Exemplo de uso
alunos = carregar_alunos()
print("Alunos carregados:", alunos)

# Adicionar um novo aluno
alunos.append({"nome": "Lucas", "nota": 9.0, "curso": "Python"})

# Salvar de volta
salvar_alunos(alunos)
print("Aluno adicionado e salvo.")

    Explicação:

    • carregar_alunos() tenta abrir o arquivo e fazer json.load. Se não existir (FileNotFoundError) ou estiver com formato inválido (JSONDecodeError), retorna uma lista vazia. Isso evita que o programa quebre.
    • salvar_alunos() recebe a lista e a sobrescreve no arquivo com json.dump.
    • Dessa forma, o programa sempre começa com os dados salvos anteriormente.

    3.6 Lidando com Erros — try , except e JSONDecodeError¶

    Ao trabalhar com arquivos, sempre podem acontecer imprevistos:

    • O arquivo pode não existir.
    • O arquivo pode estar corrompido (JSON inválido).
    • O usuário pode ter alterado o arquivo manualmente.

    Vamos criar uma função robusta:

    import json

def ler_json_seguro(caminho):
    try:
        with open(caminho, "r", encoding="utf-8") as f:
            return json.load(f)
    except FileNotFoundError:
        print(f"Arquivo {caminho} não encontrado. Criando novo.")
        return {}
    except json.JSONDecodeError:
        print(f"Erro: {caminho} não é um JSON válido. Retornando vazio.")
        return {}
    except Exception as e:
        print(f"Erro inesperado: {e}")
        return {}

    🛑 CUIDADO

    Nunca capture Exception sem necessidade. É melhor capturar exceções específicas. Mas em programas pequenos, um tratamento genérico pode ser aceitável para evitar travamentos.

    3.7 Comparação — JSON vs. Arquivo de Texto Personalizado¶

    Aspecto Texto Personalizado JSON
    Estrutura Definida por você Padrão universal
    Manutenção Difícil (parser manual) Fácil (bibliotecas prontas)
    Legibilidade Variável Boa
    Suporte a tipos complexos Precisa codificar manualmente Natvio (listas, dicionários, números, booleanos, null)
    Intercâmbio com outras linguagens Complicado Muito fácil
    Tamanho do arquivo Geralmente menor (sem formatação extra) Um pouco maior devido a chaves/colchetes

    🧠 VOCÊ SABIA?

    JSON é um subconjunto da sintaxe de objetos JavaScript, mas a maioria das linguagens tem bibliotecas para manipular JSON. Por isso, você pode compartilhar arquivos JSON entre Python, Java, C#, PHP, etc., sem problemas.

    3.8 Erros Comuns ao Trabalhar com Arquivos JSON¶

    Erro Causa Solução
    FileNotFoundError O arquivo não existe ao tentar ler. Usar try/except ou verificar com os.path.exists() .
    json.JSONDecodeError O conteúdo não é JSON válido. Validar o JSON online ou corrigir manualmente.
    PermissionError Não tem permissão para escrever na pasta. Salvar em um diretório com permissão.
    UnicodeEncodeError Tentar salvar caracteres especiais sem encoding='utf-8' . Sempre usar encoding='utf-8' ao abrir arquivos.
    Esquecer ensure_ascii=False Caracteres acentuados viram \uXXXX . Usar ensure_ascii=False no json.dump .

    3.9 Resumo do Módulo 3¶

    • json.dump(obj, arquivo) — escreve objeto Python em arquivo JSON.
    • json.load(arquivo) — lê JSON de arquivo e retorna objeto Python.
    • Use with open(...) as f: para garantir fechamento automático.
    • Sempre use encoding="utf-8" e ensure_ascii=False para suporte a caracteres especiais.
    • Trate exceções (FileNotFoundError, JSONDecodeError) para tornar seu programa robusto.
    • JSON é o formato preferido para salvar estruturas de dados complexas.

    PROJETO COMPLETO: CADASTRO DE USUÁRIOS COM JSON E TKINTER¶

    5.1 Motivação — Unindo Tudo que Aprendemos¶

    Até agora, vimos persistência, JSON e interfaces gráficas separadamente. Agora, vamos construir um projeto completo que integra todos esses conceitos em um sistema funcional e bem estruturado.

    Funcionalidades:

    • Cadastrar usuários (nome, email, idade)
    • Listar usuários
    • Buscar por nome
    • Editar dados de um usuário
    • Excluir usuário
    • Salvar automaticamente em JSON
    • Carregar dados ao iniciar

    5.2 Estrutura do Projeto¶

    Para manter o código organizado, vamos separar em três partes:

    1. Modelo de dados — funções para carregar/salvar e manipular a lista de usuários.
    2. Interface gráfica — montagem da janela e widgets.
    3. Eventos — funções que respondem a cliques.

    📌 RESUMINDO

    Essa separação é chamada de arquitetura MVC (Model-View-Controller) simplificada. O modelo (dados) é independente da view (interface).

    5.3 Código Completo Comentado¶

    import json
from tkinter import Tk, Label, Entry, Button, Listbox, Scrollbar, Toplevel, END, messagebox

# ============================
# MODELO (Dados)
# ============================

ARQUIVO_JSON = "usuarios.json"

def carregar_dados():
    """Retorna a lista de usuários do arquivo JSON."""
    try:
        with open(ARQUIVO_JSON, "r", encoding="utf-8") as f:
            return json.load(f)
    except (FileNotFoundError, json.JSONDecodeError):
        return []

def salvar_dados(usuarios):
    """Salva a lista de usuários no arquivo JSON."""
    with open(ARQUIVO_JSON, "w", encoding="utf-8") as f:
        json.dump(usuarios, f, indent=2, ensure_ascii=False)

# ============================
# INTERFACE PRINCIPAL
# ============================

class App:
    def __init__(self, janela):
        self.janela = janela
        self.janela.title("Sistema de Usuários")
        self.janela.geometry("500x400")

        # Dados
        self.usuarios = carregar_dados()

        # Widgets
        self.criar_widgets()
        self.atualizar_lista()

    def criar_widgets(self):
        # Frame para formulário
        frame_form = Frame(self.janela)
        frame_form.pack(pady=10)

        Label(frame_form, text="Nome:").grid(row=0, column=0, padx=5, pady=2)
        self.entry_nome = Entry(frame_form, width=20)
        self.entry_nome.grid(row=0, column=1, padx=5, pady=2)

        Label(frame_form, text="Email:").grid(row=1, column=0, padx=5, pady=2)
        self.entry_email = Entry(frame_form, width=20)
        self.entry_email.grid(row=1, column=1, padx=5, pady=2)

        Label(frame_form, text="Idade:").grid(row=2, column=0, padx=5, pady=2)
        self.entry_idade = Entry(frame_form, width=5)
        self.entry_idade.grid(row=2, column=1, sticky="w", padx=5, pady=2)

        # Botões
        frame_botoes = Frame(self.janela)
        frame_botoes.pack(pady=5)

        Button(frame_botoes, text="Adicionar", command=self.adicionar).grid(row=0, column=0, padx=5)
        Button(frame_botoes, text="Buscar", command=self.buscar).grid(row=0, column=1, padx=5)
        Button(frame_botoes, text="Editar", command=self.editar).grid(row=0, column=2, padx=5)
        Button(frame_botoes, text="Excluir", command=self.excluir).grid(row=0, column=3, padx=5)
        Button(frame_botoes, text="Limpar campos", command=self.limpar_campos).grid(row=0, column=4, padx=5)

        # Lista com scroll
        frame_lista = Frame(self.janela)
        frame_lista.pack(fill="both", expand=True, padx=10, pady=10)

        scrollbar = Scrollbar(frame_lista)
        scrollbar.pack(side="right", fill="y")

        self.lista = Listbox(frame_lista, yscrollcommand=scrollbar.set)
        self.lista.pack(side="left", fill="both", expand=True)
        scrollbar.config(command=self.lista.yview)

        # Vincular duplo clique para editar
        self.lista.bind("<Double-Button-1>", self.editar_selecionado)

    # ============================
    # MÉTODOS AUXILIARES
    # ============================

    def atualizar_lista(self, filtro=None):
        """Atualiza a Listbox com os usuários."""
        self.lista.delete(0, END)
        lista = self.usuarios if filtro is None else filtro
        for idx, usuario in enumerate(lista):
            self.lista.insert(END, f"{usuario['nome']} - {usuario['email']}")

    def limpar_campos(self):
        """Limpa os campos de entrada."""
        self.entry_nome.delete(0, END)
        self.entry_email.delete(0, END)
        self.entry_idade.delete(0, END)

    def pegar_indice_selecionado(self):
        """Retorna o índice do item selecionado na lista."""
        selecionado = self.lista.curselection()
        if not selecionado:
            messagebox.showwarning("Aviso", "Selecione um usuário na lista.")
            return None
        return selecionado[0]

    # ============================
    # CRUD
    # ============================

    def adicionar(self):
        nome = self.entry_nome.get().strip()
        email = self.entry_email.get().strip()
        idade = self.entry_idade.get().strip()

        # Validações
        if not nome:
            messagebox.showerror("Erro", "O nome é obrigatório.")
            return
        if not email:
            messagebox.showerror("Erro", "O email é obrigatório.")
            return
        if not idade.isdigit() or int(idade) <= 0:
            messagebox.showerror("Erro", "Idade inválida. Digite um número positivo.")
            return

        # Verificar email duplicado
        for u in self.usuarios:
            if u["email"].lower() == email.lower():
                messagebox.showerror("Erro", "Este email já está cadastrado.")
                return

        # Adicionar
        novo = {"nome": nome, "email": email, "idade": int(idade)}
        self.usuarios.append(novo)
        salvar_dados(self.usuarios)
        self.atualizar_lista()
        self.limpar_campos()
        messagebox.showinfo("Sucesso", "Usuário cadastrado!")

    def buscar(self):
        termo = self.entry_nome.get().strip().lower()
        if not termo:
            self.atualizar_lista()
            return
        filtrados = [u for u in self.usuarios if termo in u["nome"].lower()]
        if filtrados:
            self.atualizar_lista(filtrados)
        else:
            messagebox.showinfo("Busca", "Nenhum usuário encontrado.")
            self.atualizar_lista()

    def editar_selecionado(self, event=None):
        """Abre janela de edição para o item selecionado."""
        indice = self.pegar_indice_selecionado()
        if indice is None:
            return
        usuario = self.usuarios[indice]
        self.abrir_janela_edicao(indice, usuario)

    def editar(self):
        """Chamado pelo botão Editar."""
        self.editar_selecionado()

    def abrir_janela_edicao(self, indice, usuario):
        """Cria uma nova janela para editar os dados."""
        janela_edicao = Toplevel(self.janela)
        janela_edicao.title("Editar Usuário")
        janela_edicao.geometry("300x200")

        Label(janela_edicao, text="Nome:").grid(row=0, column=0, padx=5, pady=5)
        entry_nome = Entry(janela_edicao)
        entry_nome.grid(row=0, column=1, padx=5, pady=5)
        entry_nome.insert(0, usuario["nome"])

        Label(janela_edicao, text="Email:").grid(row=1, column=0, padx=5, pady=5)
        entry_email = Entry(janela_edicao)
        entry_email.grid(row=1, column=1, padx=5, pady=5)
        entry_email.insert(0, usuario["email"])

        Label(janela_edicao, text="Idade:").grid(row=2, column=0, padx=5, pady=5)
        entry_idade = Entry(janela_edicao)
        entry_idade.grid(row=2, column=1, padx=5, pady=5)
        entry_idade.insert(0, str(usuario["idade"]))

        def salvar_edicao():
            novo_nome = entry_nome.get().strip()
            novo_email = entry_email.get().strip()
            nova_idade = entry_idade.get().strip()

            if not novo_nome or not novo_email or not nova_idade.isdigit():
                messagebox.showerror("Erro", "Preencha todos os campos corretamente.")
                return

            # Verificar email duplicado (excluindo o próprio)
            for i, u in enumerate(self.usuarios):
                if i != indice and u["email"].lower() == novo_email.lower():
                    messagebox.showerror("Erro", "Este email já está em uso.")
                    return

            # Atualizar
            self.usuarios[indice] = {"nome": novo_nome, "email": novo_email, "idade": int(nova_idade)}
            salvar_dados(self.usuarios)
            self.atualizar_lista()
            janela_edicao.destroy()
            messagebox.showinfo("Sucesso", "Dados atualizados!")

        Button(janela_edicao, text="Salvar", command=salvar_edicao).grid(row=3, column=0, columnspan=2, pady=10)

    def excluir(self):
        indice = self.pegar_indice_selecionado()
        if indice is None:
            return
        usuario = self.usuarios[indice]
        confirmar = messagebox.askyesno("Confirmar", f"Deseja excluir {usuario['nome']}?")
        if confirmar:
            self.usuarios.pop(indice)
            salvar_dados(self.usuarios)
            self.atualizar_lista()
            self.limpar_campos()
            messagebox.showinfo("Sucesso", "Usuário excluído!")

# ============================

if __name__ == "__main__":
    root = Tk()
    app = App(root)

    5.4 Explicação do Projeto¶

    • Modelo: As funções carregar_dados e salvar_dados isolam o acesso ao arquivo JSON.
    • Classe App: Encapsula a lógica da interface e os dados. O uso de uma classe facilita o compartilhamento de variáveis (como self.usuarios) entre os métodos.
    • Validações robustas: Verificação de campos vazios, formato de idade, email duplicado.
    • Edição: Abre uma nova janela (Toplevel) para editar os dados, evitando sobrecarregar a janela principal.
    • Exclusão com confirmação: Usa messagebox.askyesno para evitar exclusões acidentais.
    • Atualização automática: Após qualquer operação, a lista é atualizada e o arquivo JSON é salvo.

    💡 DICA DE OURO

    Sempre que modificar os dados (adicionar, editar, excluir), salve imediatamente no arquivo. Assim, se o programa travar, os dados já estarão persistidos.
