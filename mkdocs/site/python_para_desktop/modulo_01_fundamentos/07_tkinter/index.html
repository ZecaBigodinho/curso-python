    ARQUITETURA PREPARADA PARA TKINTER¶

    4.1 Motivação — Por que o Terminal é Limitado¶

    Até agora, todos os programas que escrevemos rodam no terminal (linha de comando). As interações são feitas com input() e print(). Isso é ótimo para aprender, mas limita a experiência do usuário.

    • input() bloqueia o programa até o usuário digitar algo e apertar Enter.
    • print() exibe texto de forma sequencial, não permite botões, campos de texto, menus visuais.

    Quando você criar uma interface gráfica (com Tkinter, PyQt, etc.), o paradigma muda completamente:

    • O programa fica em execução o tempo todo, esperando eventos (cliques em botões, teclas pressionadas).
    • Não há um fluxo linear de entrada/saída; o programa reage a interações do usuário.

    Analogia do Garçom vs. Self-Service

    • No terminal, você é o cliente que pede (input) e recebe (print) — é como um garçom.
    • Na interface gráfica, o programa é o restaurante que oferece um cardápio (botões, menus) e você escolhe livremente o que fazer — é como um self-service.

    4.2 Explicação — Eventos e Funções Callback¶

    Em uma interface gráfica, o programa entra em um loop de eventos (event loop). Ele fica "escutando" o que o usuário faz. Quando um evento acontece (ex.: clique em um botão), uma função callback é chamada.

    Estrutura típica:

    from tkinter import *

def ao_clicar():
    print("Botão clicado!")

# Cria a janela
janela = Tk()
botao = Button(janela, text="Clique aqui", command=ao_clicar)
botao.pack()
janela.mainloop()   # Inicia o loop de eventos

    Perceba:

    • A função ao_clicar não é chamada por nós; ela é registrada para ser chamada quando o botão for clicado.
    • janela.mainloop() é um loop infinito que mantém a janela aberta e processa eventos.

    Comparação com o que você já sabe:

    Conceito Terminal (while True com input) Interface Gráfica (Tkinter)
    Controle de fluxo Linear (você controla o loop) Baseado em eventos (o sistema controla)
    Entrada input() (bloqueante) Cliques, teclas (não bloqueantes)
    Saída print() (texto) Widgets (labels, textos, imagens)
    Funções Chamadas diretamente Chamadas como callback (por eventos)

    4.3 Exemplo Pequeno — Um Botão que Salva um Arquivo¶

    Vamos criar uma janela com um botão que, ao ser clicado, salva um arquivo JSON com um dado fixo.

    import json
from tkinter import Tk, Button

def salvar_json():
    dados = {"mensagem": "Salvo pelo botão!"}
    with open("botao.json", "w", encoding="utf-8") as f:
        json.dump(dados, f, indent=2, ensure_ascii=False)
    print("Arquivo salvo!")

janela = Tk()
janela.title("Salvar JSON")

botao = Button(janela, text="Salvar JSON", command=salvar_json)
botao.pack(padx=20, pady=20)

janela.mainloop()

    Explicação:

    • command=salvar_json — associa a função salvar_json ao evento de clique.
    • Quando o botão for pressionado, a função será executada.
    • O arquivo será criado na mesma pasta do script.

    💡 DICA DE OURO

    Em Tkinter, a função callback não deve receber argumentos (a menos que você use técnicas avançadas com lambda ). Por isso, toda a lógica deve estar dentro da função.

    4.4 Exemplo Intermediário — Campos de Entrada e Validação¶

    Agora, vamos criar uma janela com dois campos (nome e idade) e um botão que salva os dados em JSON.

    import json
from tkinter import Tk, Label, Entry, Button

def salvar():
    nome = entrada_nome.get()
    idade = entrada_idade.get()

    # Validação simples
    if not nome.strip():
        print("Nome não pode estar vazio!")
        return
    if not idade.isdigit():
        print("Idade deve ser um número!")
        return

    dados = {"nome": nome, "idade": int(idade)}
    with open("cadastro.json", "w", encoding="utf-8") as f:
        json.dump(dados, f, indent=2, ensure_ascii=False)
    print("Cadastro salvo com sucesso!")

janela = Tk()
janela.title("Cadastro")

Label(janela, text="Nome:").grid(row=0, column=0, padx=5, pady=5)
entrada_nome = Entry(janela)
entrada_nome.grid(row=0, column=1, padx=5, pady=5)

Label(janela, text="Idade:").grid(row=1, column=0, padx=5, pady=5)
entrada_idade = Entry(janela)
entrada_idade.grid(row=1, column=1, padx=5, pady=5)

Button(janela, text="Salvar", command=salvar).grid(row=2, column=0, columnspan=2, pady=10)

janela.mainloop()

    Explicação da validação:

    • entrada_nome.get() obtém o texto do campo.
    • strip() remove espaços extras.
    • isdigit() verifica se a string é composta apenas por dígitos.

    🛑 CUIDADO

    Nunca confie nos dados do usuário. Sempre valide antes de salvar. Senão, você pode acabar com dados inconsistentes (ex.: idade vazia, nome com números, etc.).

    4.5 Exemplo Real — Sistema de Cadastro com JSON e Tkinter¶

    Vamos construir um sistema completo que: Permite adicionar, listar e excluir pessoas. Persiste os dados em um arquivo JSON. Tudo em uma janela gráfica.

    import json
from tkinter import Tk, Label, Entry, Button, Listbox, Scrollbar, END

ARQUIVO = "pessoas.json"

def carregar():
    try:
        with open(ARQUIVO, "r", encoding="utf-8") as f:
            return json.load(f)
    except (FileNotFoundError, json.JSONDecodeError):
        return []

def salvar(dados):
    with open(ARQUIVO, "w", encoding="utf-8") as f:
        json.dump(dados, f, indent=2, ensure_ascii=False)

def atualizar_lista():
    lista_pessoas.delete(0, END)
    for p in pessoas:
        lista_pessoas.insert(END, f"{p['nome']} - {p['idade']} anos")

def adicionar():
    nome = entrada_nome.get().strip()
    idade = entrada_idade.get().strip()
    if not nome:
        print("Nome obrigatório!")
        return
    if not idade.isdigit():
        print("Idade deve ser número!")
        return
    pessoas.append({"nome": nome, "idade": int(idade)})
    salvar(pessoas)
    atualizar_lista()
    entrada_nome.delete(0, END)
    entrada_idade.delete(0, END)

def excluir():
    selecionado = lista_pessoas.curselection()
    if not selecionado:
        print("Selecione uma pessoa!")
        return
    indice = selecionado[0]
    removido = pessoas.pop(indice)
    salvar(pessoas)
    atualizar_lista()
    print(f"{removido['nome']} removido.")

# Interface
janela = Tk()
janela.title("Cadastro de Pessoas")

Label(janela, text="Nome:").grid(row=0, column=0)
entrada_nome = Entry(janela)
entrada_nome.grid(row=0, column=1)

Label(janela, text="Idade:").grid(row=1, column=0)
entrada_idade = Entry(janela)
entrada_idade.grid(row=1, column=1)

Button(janela, text="Adicionar", command=adicionar).grid(row=2, column=0, columnspan=2, pady=5)

Label(janela, text="Lista de Pessoas:").grid(row=3, column=0, columnspan=2)

scrollbar = Scrollbar(janela)
scrollbar.grid(row=4, column=2, sticky="ns")

lista_pessoas = Listbox(janela, yscrollcommand=scrollbar.set, height=10)
lista_pessoas.grid(row=4, column=0, columnspan=2, padx=5)
scrollbar.config(command=lista_pessoas.yview)

Button(janela, text="Excluir selecionado", command=excluir).grid(row=5, column=0, columnspan=2, pady=5)

# Carregar dados iniciais
pessoas = carregar()
atualizar_lista()

janela.mainloop()

    Explicação detalhada do código:

    1. carregar() e salvar() gerenciam o arquivo JSON.
    2. atualizar_lista() limpa a Listbox e reinsere todos os nomes.
    3. adicionar() valida os campos, adiciona à lista, salva e atualiza.
    4. excluir() usa curselection() para obter o índice selecionado na Listbox, remove da lista, salva e atualiza.
    5. A interface usa grid() para organizar os widgets.

    🧠 VOCÊ SABIA?

    A função curselection() retorna uma tupla com os índices dos itens selecionados. Mesmo que você possa selecionar apenas um, é sempre uma tupla.

    4.6 Como Adaptar Seu Código do Terminal para Interface Gráfica¶

    Se você já tem um sistema no terminal com menu (while True e input), precisa reestruturar:

    • Antes: Um loop que pergunta "1 - Cadastrar, 2 - Listar, ..." e chama funções.
    • Depois: Cada opção vira um botão (ou menu), e cada botão chama uma função.

    Exemplo de adaptação:

    Código antigo (terminal):

    def cadastrar():
    nome = input("Nome: ")
    # ...

while True:
    opcao = input("1 - Cadastrar, 0 - Sair: ")
    if opcao == "1":
        cadastrar()
    elif opcao == "0":
        break

    Código novo (Tkinter):

    def cadastrar():
    nome = entrada_nome.get()   # Em vez de input()
    # ...

# Em vez do loop, criamos botões:
Button(janela, text="Cadastrar", command=cadastrar).pack()
Button(janela, text="Sair", command=janela.quit).pack()

    💡 DICA DE OURO

    Separe a lógica de negócio (cadastrar, listar, salvar, carregar) da interface. Assim, você pode trocar terminal por Tkinter sem reescrever toda a lógica. Mantenha funções como cadastrar_pessoa(nome, idade) que recebem parâmetros, em vez de dependerem de input() .

    4.7 Erros Comuns ao Migrar para Tkinter¶

    Erro Causa Solução
    Usar input() em interface gráfica input() não funciona em GUI. Substituir por Entry e get() .
    Esquecer mainloop() A janela não aparece. Adicionar janela.mainloop() .
    Tentar passar argumentos para command command espera uma função sem argumentos. Usar lambda ou criar funções sem parâmetros. Exemplo: command=lambda: minha_funcao(arg) .
    Bloquear a interface com loops longos A interface trava. Nunca coloque time.sleep ou loops longos no código do evento; use after() para tarefas assíncronas.
    Não validar dados Dados inválidos são salvos. Sempre valide antes de salvar.

    4.8 Resumo do Módulo 4¶

    • Interface gráfica usa eventos e callbacks, não input()/print().
    • command associa um botão a uma função.
    • Valide sempre os dados do usuário antes de persistir.
    • Separe a lógica de negócio da interface para facilitar manutenção.
    • Use Listbox para exibir dados e curselection() para selecionar itens.
    • O mainloop() mantém a janela ativa.
